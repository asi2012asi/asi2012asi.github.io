破解第一个应用程序

一般破解程序可以从错误信息入手 例如toast提示 注册失败！ 那么可以在资源文件中找到该string字段 通过R.java文件找到对应的id 在反编译后的smali文件中查找该id 读dalvik指令集 判断注册成功的地方 修改代码 重新打包dex

工具 apktool 编译和反编译工具 在windows或者Ubuntu下可用
windows下使用方法

  1. 反编译 apktool d abc.apk outdir
  2. 编译 apktool b app路径 abc2.apk

工具 signapk.jar 签名工具 位于android源码目录下/build/tools/signapk/SignApk.java 可使用Android系统提供的签名文件进行签名testkey.pk8和testkey.x509.pem

新建批处理文件signapk.bat

java -jar "%~dp0signapk.jar" "%~dp0testkey.x509.pem" "%~dp0testkey.pk8" %1signed.apk

将testkey.pk8和testkey.x509.pem和signapk.jar,signapk.bat放在同一目录下 cmd进入该目录 运行signapk abc.apk 就会在该目录下生产apk文件

dex文件反汇编工具 主流的有bakSmali和Dedexer
将dex文件反汇编成smali文件 采用baksmali的代码示例为
java -jar baksmali.jar -o baksmaliout Hello.dex

编译smali文件使用smali.jar 示例java -jar smali.jar -o classes.dex HelloWorld.smali

了解Android Dalvik虚拟机

以下简写java虚拟机为jvm Android dalvik虚拟机为dvm

jvm和dvm的区别

  1. jvm运行的是java字节码即class文件 dvm运行的是dalvik字节码 jvm通过解码class文件运行程序 而dvm是通过dx工具将class转为dalvik字节码
  2. dalvik可执行文件体积更小 在转换过程中会对class重排列 消除冗余信息减少dalvik可执行文件的体积
  3. jvm和dvm架构不同 jvm是基于栈架构 运行时jvm需要频繁从栈上读取和写入数据这个过程需要更多指令分派和内存访问 会消耗不少cpu时间 而dvm是基于寄存器架构的 数据访问通过寄存器间直接传递

dvm是如何执行程序的

dvm属于Android Runtime层 从Android系统启动开始分析dvm是如何执行程序的

  1. Android加载完Linux内核后 第一个执行的进程是init进程
  2. 运行init.rc脚本启动zygote进程 zygote进程是进程孵化器
  3. zygote进程在启动过程中 先初始化dvm 然后启动system_server进入zygote模式
  4. 创建出一个server socket等待activityManagerService向zygote发送一个进程间通信请求 来创建一个新的应用进程
  5. zygote收到请求后通过fork自身来创建一个dvm实例来执行main函数 fork成功后工作就交给dvm

下面是dvm的执行过程

  1. 首先通过LoadclassfromDex函数完成类的装载工作
  2. 虚拟机使用gDvm.loadedClasses来存储和查询所有装载进来的类
  3. 字节码验证器使用dvmVerifyCodeFlow对装入的代码进行校验
  4. 虚拟机调用finaClass查找并装载main函数
  5. 调用dvmInterpret初始化解释器并执行字节码流

关于jit

即时编译 从Android2.2开始加入 是一种通过在运行时将字节码翻译成机械码的技术 主流jit包含两种字节码编译方式

  1. method方式 以函数为单位进行编译
  2. trace方式 以trace为单位进行编译 trace分热路径和冷路径 很少被执行的称为冷路径 频繁执行的称为热路径

虚拟机默认trace方式编译代码

dalvik指令集

  1. 空操作指令 nop
  2. 数据操作指令 例如move vA,vB 就是将vB寄存器的值赋给vA寄存器 两种寄存器都是4位
  3. 返回指令 例如return-void 函数从一个void方法返回
  4. 数据定义指令 例如 const/4 vA,#+B 将数值符号扩展为32位后赋给寄存器vA
  5. 锁指令 例如monitor-enter vAA 为指定的对象获取锁
  6. 实例操作指令 例如new-instance vAA,type@BBBB 构造一个指定类型对象的新实例 并将对象引用赋值给vAA寄存器 类型符type指定的类型不能是数组类
  7. 数据操作指令 例如array-length vA,vB 获取给定vB寄存器中数组的长度并将值赋给vA寄存器 数组长度指的是数组的条目个数
  8. 异常指令 throw vAA 抛出vAA寄存器中指定类型的异常
  9. 跳转指令 goto +AA 无条件跳转到指定偏移处 偏移量AAAA不能为0
  10. 比较指令 cmpl-float 比较两个单精度浮点数 如果vBB寄存器大于vCC寄存器 则结果为-1 相等为0 小于为1
  11. 字段操作指令 普通字段操作指令有iget iget-wide 静态字段操作指令sget sget-wide
  12. 方法调用指令 invoke-virtual 或invoke-virtual/range 调用实例虚方法
  13. 数据转换指令 float-to-int 将单精度浮点型转为整形
  14. 数据运算指令 add-int v1,v2 将v1寄存器与v2寄存器的值相加
